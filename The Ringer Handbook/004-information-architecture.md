# Information Architecture

## Executive Summary

Information Architecture defines how players understand Ringer. Every screen should answer one primary question.

## The Five Primary Jobs

1. Discover — I want to play.
2. Network — I want to know who I play with.
3. Games — I want to manage my games.
4. Inbox — I need information about active games.
5. Profile — I want to manage my sporting identity.

## Primary Navigation

- Discover
- Network
- Games
- Inbox
- Profile

## Discover

Purpose: Help players find their next opportunity.

Contents:
- Recommended Games
- Nearby Games
- Games Your Network Is Playing
- Suggested Communities
- Suggested Connections

Primary Action: Join Game

## Network

Purpose: Build and maintain your sporting community.

Contents:
- Connections
- Pending Requests
- Suggested Connections
- Mutual Connections
- Communities

Primary Action: Connect

## Games

Purpose: Manage games you've created or joined.

Contents:
- Upcoming
- Hosting
- Invited
- History

Primary Action: Create Game

## Inbox

Purpose: Coordinate games already in progress.

Contents:
- Game Chats
- Direct Messages
- Invitations
- System Messages

Primary Action: Reply

## Profile

Purpose: Represent your sporting identity.

Contents:
- Sports
- Skill Levels
- Availability
- Reputation
- Playing History
- Payment Settings

Primary Action: Edit Profile

## Navigation Principles

- Every screen has one obvious action.
- Primary navigation never changes.
- Never hide core features.
- Important actions are no more than three taps away.
- Context determines available actions.

## Acceptance Criteria

- Every feature maps to one primary navigation item.
- Bottom navigation remains limited to five destinations.
- Discover is the default landing experience.
